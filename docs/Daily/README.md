# Daily plans

Generated by the `morning` / `plan` / `report` / `晨报` trigger.

One file per day: `YYYY-MM-DD.md` (or `YYYY-MM-DD-<slot>.md` if you run the trigger twice in a day — e.g. `-morning`, `-afternoon`).

## What lives here
- Prioritized work for today, synthesized from:
  - `docs/note for myself.md` (the raw inbox)
  - Yesterday's daily plan (carryover)
  - Recent Journal entries (context)
  - `docs/Peako_Open_Decisions.md` (open product calls)
  - Recent git commits (what changed while you were away)
- Top 3, secondary items, open questions, stale inbox items, and carryover

## What does NOT live here
- Product logic → `Logic/`
- Decisions with rationale → `Decisions/`
- Session summaries → `Journal/`

Daily plans are **operational** (what I'm doing today), not **conceptual** (what Peako is). They're tagged `#daily-plan` so Dataview can keep them separate from the knowledge base.

## Template
Copy `Templates/Daily.md`. The agent does this for you automatically.
